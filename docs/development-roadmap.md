# 开发路线图

## 阶段一：规划
- 扫描仓库和技术栈。
- 输出产品规划、信息架构、页面地图、用户流程、设计系统、组件清单、数据模型、动效规范、响应式规范。
- 确定 MVP 边界。

## 阶段二：前端原型
- React + Vite + TypeScript 初始化。
- Tailwind CSS 设计 Token。
- React Router 懒加载路由。
- 本地模拟数据。
- 首页、成员、活动、签到、社区、动态、个人中心、登录注册、后台框架、404。
- 覆盖加载、空、错误、未登录、无权限状态。

## 阶段三：Supabase 接入
- Supabase Auth 注册、登录、登出、找回密码。
- Profile、Member、Event、OfficialUpdate 数据读取。
- Storage 上传图片/视频。
- 基础 RLS 和权限策略。

## 阶段四：签到与社区
- 服务端签到函数。
- CheckIn、PointLedger、Badge。
- 发帖、编辑、删除、点赞、评论、收藏、举报、通知。

## 阶段五：后台和审核
- 成员、活动、动态管理。
- 社区审核、评论审核、举报处理。
- 用户禁言/封禁。
- 积分和徽章规则。
- AdminLog。

## 阶段六：测试和优化
- TypeScript、lint、构建。
- 权限与 RLS 测试。
- 签到并发和重复提交测试。
- 上传文件测试。
- 移动端、桌面端、减少动态效果、SEO 和可访问性检查。
