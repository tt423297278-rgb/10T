# 产品需求文档

## 结论
第一版建设目标是一个“十个勤天粉丝陪伴型社区”的 React + Vite SPA 原型：用户可以浏览成员、查看活动、每日签到、阅读成员动态、参与轻社区互动，并看到个人中心和后台管理框架。真实数据库、RLS、Edge Function 和审核链路在后续阶段接入。

## 定位
- 产品一句话：记录十位成员成长、整理成员活动、支持粉丝签到和互动交流的温暖陪伴型社区。
- 关键词：麦田、土地、成长、青年、陪伴、自然、生命力、共同生长、轻手册、纪实感、温暖但克制。
- 风格边界：不做廉价饭圈、游戏积分面板、低幼卡通、传统论坛和模板企业官网。

## 目标用户
- 核心用户：长期关注十个勤天的粉丝，想查资料、看活动、签到记录陪伴。
- 内容贡献者：愿意发布活动记录、摄影、观后感和成长记录的粉丝。
- 运营管理员：维护成员资料、活动、动态、审核社区内容。

## MVP 范围
- 首页：十位成员群像 Hero、成员入口、最近活动、今日签到、成员动态、社区精选、成长数字、页脚。
- 成员：十位成员列表、详情页、统一本地数据源、占位资料明确标注。
- 活动：月历/列表、移动端时间线、成员/类型/状态筛选、活动详情和 ICS 预留。
- 食堂：基于获授权的禾伙人全国巡吃表，按省份/地区、城市、餐厅名称关键词、食物分类和可多选人均价格区间筛选餐厅，筛选状态写入 URL；展示价格、地址、到店提醒与原表来源；登录用户可在确认实际到店后按口味、服务、性价比、环境进行 0.5–5 星评分，选填最多 120 字的一句话点评，并上传最多 4 张到店图片，一人一店只保留一份评价；登录用户也可推荐新餐厅，名称、城市、分类、四项评分和实际到店确认均为必填，详细地址允许留空或通过高德地图 POI 搜索选取，投稿审核通过前不公开。
- 签到：本地模拟每日签到、连续/累计天数、麦粒值明细、重复签到状态。
- 社区：帖子列表、详情、发布入口状态、点赞/评论/收藏的模拟交互状态。
- 动态：官方公开动态和粉丝帖分离。
- 个人中心：资料、签到、麦粒值、内容、关注、提醒、徽章。
- 后台：管理框架和模拟数据，暂不实现完整业务写入。

## 非 MVP
私信、复杂等级、积分商城、实物兑换、抽奖、直播、自动抓取第三方平台、复杂推荐、多级会员、复杂补签、原生 App。

## 关键约束
- 未确认资料不虚构，使用“待补充/待核验”占位。
- 不热链第三方素材，不搬运受版权保护视频。
- 首页首屏必须以十位成员群像为视觉核心；没有授权合照前只能使用明确标注的十人剪影占位，不生成 AI 人脸、不抓取第三方图片。
- 签到和积分最终必须服务端校验，前端模拟仅用于原型。
- 普通用户不能进入后台，管理员权限最终必须由数据库和服务端校验。

## 阶段一发现
- 当前仓库为空，已按 React + Vite + TypeScript 初始化。
- 系统 PATH 无 node/npm/git，使用 Codex 内置 Node、pnpm、Git 运行。
- UI UX Pro Max 原始推荐偏“活力块面/蓝色社区”，与本项目“麦田自然、温暖克制”冲突；最终设计系统保留其可用性建议，视觉 Token 调整为麦田手册方向。
