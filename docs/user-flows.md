# 用户流程

## 新用户首次进入
1. 进入首页，看到 Hero、今日活动和签到入口。
2. 点击成员入口查看十位成员。
3. 点击活动查看近期安排。
4. 尝试签到、收藏或发帖时进入登录提示。

## 注册与登录
1. 注册页填写昵称、邮箱、密码。
2. React Hook Form + Zod 进行前端校验。
3. 阶段二为模拟提交；阶段三接 Supabase Auth。
4. 成功后返回来源页面，并显示全局提示。

## 签到
1. 用户进入 `/check-in`。
2. 未登录时提示登录。
3. 登录后点击签到。
4. 前端原型模拟今日唯一签到、连续天数、累计天数和麦粒值明细。
5. 后续使用数据库唯一约束、RPC 或 Edge Function 防止重复和并发。

## 活动筛选
1. 用户进入活动页。
2. 默认显示当月月历和近期列表。
3. 可按成员、类型、状态切换筛选。
4. 移动端展示时间线，桌面端展示月历 + 列表双栏。
5. 点击活动进入详情，查看来源和提醒入口。

## 食堂筛选与评分
1. 用户进入 `/canteen`，先选择省份/地区，再按城市和食物分类缩小范围。
2. 人均价格支持多选，多个区间按“任一命中”匹配；“价格不详”可独立筛选，筛选状态写入 URL。
3. 餐厅卡片展示综合评分和评价人数；点击后打开四维评分弹窗。
4. 未登录用户进入登录页，成功后返回原食堂筛选地址；未配置 Supabase 时只展示不可用提示，不提供假提交。
5. 登录用户按口味、服务、性价比、环境分别选择 0.5–5 星，并确认自己实际到店消费。
6. 首次提交创建评分；再次提交更新原评分。数据库唯一约束保证一人一店只计一份。

## 社区发布与互动
1. 用户进入社区页浏览精选、最新、热门。
2. 点击发布，未登录时出现登录提示。
3. 登录用户填写标题、正文、分类、关联成员、上传媒体。
4. 上传/发布中显示进度；失败保留输入内容。
5. 内容可进入审核中、已发布、隐藏或删除状态。

## 管理员审核
1. 管理员进入 `/admin`。
2. 查看内容队列、举报、用户状态和活动维护。
3. 审核动作写入 AdminLog。
4. 阶段二仅提供页面框架；阶段五接真实权限和审计。
