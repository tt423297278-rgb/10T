# 信息架构

## 结论
信息架构采用“公开内容 + 登录陪伴 + 运营后台”三层结构。公开页面负责浏览和理解，登录页面负责参与和记录，后台负责长期运营。

## 顶层导航
- 首页：社区概览和今日入口。
- 成员：十位成员资料、详情、成长节点。
- 活动：日历、列表、筛选、提醒。
- 食堂：城市、食物分类、餐厅记录、开饭签和原表来源。
- 签到：每日陪伴、连续记录、麦粒值。
- 社区：粉丝发布、互动、收藏。
- 动态：官方公开信息。
- 个人中心：用户资料、内容、通知、徽章。
- 后台：管理员内容维护和审核。

## 内容分层
- 成员资料：Member、MemberTimeline、MemberWork、MemberMedia。
- 活动信息：Event、EventMember、EventReminder。
- 签到成长：CheckIn、PointLedger、Badge、UserBadge。
- 社区互动：Post、PostMedia、Comment、Like、Favorite、Report。
- 官方动态：OfficialUpdate、OfficialUpdateMember。
- 管理审计：AdminLog。

## 权限分层
- 游客：浏览公开页面，不能签到、发布、点赞、收藏、评论。
- 登录用户：可签到、发帖、评论、收藏、关注成员、管理自己的内容。
- 管理员：可进入后台，管理成员、活动、动态、社区审核、用户状态、规则。

## 移动端优先
移动端使用底部导航承载核心入口：首页、成员、活动、签到、社区。个人中心和后台保留在头像菜单或页面入口中。
