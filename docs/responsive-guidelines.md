# 响应式规范

## 结论
移动端优先，桌面端增强信息密度。日历、社区和后台不能简单拉伸移动布局。

## 断点
- `sm`: 640px。
- `md`: 768px。
- `lg`: 1024px。
- `xl`: 1280px。

## 移动端
- 使用底部导航，最多 5 个主入口。
- 日历展示为时间线和紧凑日期条。
- 成员卡片单列或横向轻滑动，不出现横向页面滚动。
- 发布、签到等高频操作在拇指可触达区域。
- 弹窗最大高度不超过视口，内容可滚动。

## 桌面端
- 首页使用 Hero + 内容区分层。
- 成员详情用主内容 + 侧栏。
- 活动页用月历 + 列表双栏。
- 社区用主 feed + 侧栏筛选/规则。
- 后台使用侧栏导航和表格/队列布局。

## 可访问性
- 最小正文 16px。
- 点击目标不小于 44px。
- 焦点样式清晰。
- 图标按钮必须有 `aria-label`。
- 所有图片需要 `alt`。
